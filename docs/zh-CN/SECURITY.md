# 安全策略

[English](../../SECURITY.md) | 简体中文

## 支持版本

安全修复优先落在 `main` 和最新发布版本；旧版本可能需要先升级后才能获得修复。

## 报告安全漏洞

请优先使用 GitHub 仓库的 **Security → Report a vulnerability** 私密报告入口。不要在公开 Issue 中提交利用细节、私有仓库路径、源码、review URL、访问令牌或未经脱敏的日志与截图。

如果私密报告入口尚未启用，可以先创建一个不含敏感细节的 Issue，仅说明需要建立私密沟通渠道。

报告中建议包含：

- 受影响版本或 commit
- 操作系统、Node.js 和浏览器版本
- 最小复现步骤与安全影响
- 已完成脱敏的日志或截图
- 可行的缓解方式（如有）

如果凭据已经进入 commit，应先撤销或轮换，再处理仓库内容。后续新增一个删除 commit 并不够，因为原值仍在 Git 历史中。任何历史重写和 force-push 都应先通过私密渠道协调。

## 安全边界

以下行为属于安全问题的重点范围：

- 非本机来源可以访问审查服务或绕过随机路径令牌
- 路径穿越、任意文件读取或写入目标仓库真实 Git index
- Diff、Markdown 或 Mermaid 内容导致脚本执行
- review URL、源码、已读状态或运行时日志意外泄露
- 缓存失效错误导致不同仓库或版本之间串读内容
